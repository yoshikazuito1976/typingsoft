
# TypingSoft（学内タイピング練習用）

本リポジトリは、学内で使用するタイピング練習用Webアプリの
開発・改善・情報共有を目的としています。

## 目的
- タイピングソフトの品質改善
- 学内職員間での仕様・課題・改善案の共有
- GitHubを用いた開発フローの学内練習

## 対象外（このリポジトリに置かないもの）
- 学生・職員の個人情報
- 学内限定資料や内部事情の詳細
- サーバ設定情報、認証情報、鍵、IPアドレス
  

ブラウザ上で動作する日本語タイピング練習ソフトです。  
現在は **入力制御ロジックの改修を進行中** です。

---

## 開発方針（重要）

- `main` ブランチは **常に動作する状態** を保つ
- 直接 `main` には push しない
- 各自の作業は **人別ブランチ** で行う
  - 例：
    - `dev/yito`
    - `dev/partner`
- `main` への反映は **Pull Request 経由** とする
---
## Issueについて
Issueは「気づきメモ・相談・改善要望」を残す場所です。
完璧な文章や技術的な説明は不要です。

### タイトル
【Bug】 / 【Improve】 / 【Note】 を先頭につけてください。

例：
【Bug】ミス入力時に文字が進んでしまう  
【Improve】初心者向けに表示を分かりやすくしたい  
【Note】ローマ字揺らぎ対応は後回しでよさそう  

### 本文（以下のうち、書けるところだけでOK）

■ 現象・気づき  
（今どうなっているか、気づいたこと）

■ 期待・要望  
（こうなってほしい、こうだと助かる）

■ 補足・背景（任意）  
（なぜそう思ったか、授業や利用の文脈など）

※短文で問題ありません。
※技術的な用語が分からなくても大丈夫です。

---

## 現在の作業内容

- 入力処理の整理
  - ミス時に入力を進めない制御
  - 正解文字のみを受け付ける方式に変更中
- ローマ字入力の揺らぎ対応は **後続フェーズ** とする

---


## ファイル構成（抜粋）

- `index.html`  
  画面構成・DOM定義
- `program.js`  
  タイピング処理・入力制御ロジック（主に改修対象）
- `kanaroma.js`  
  カナ → ローマ字対応表
- `style.css`  
  表示・配色・レイアウト

---

## 動作確認方法（ローカル）

Apache 等で `index.html` を配信し、ブラウザからアクセスしてください。
